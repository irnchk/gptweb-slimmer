ChatGPT 대화가 길어지면 **지난 턴이 모두 페이지에 남아** 브라우저가 계속 다시 그리기 때문에 느려집니다. 이 크롬 확장 프로그램은 최신 **N개** 턴만 DOM에 남기고 나머지는 페이지에서 분리해, 메시지나 서버는 건드리지 않으면서 스크롤·입력·렌더링을 쾌적하게 유지합니다.

> 💡 이 확장은 서버 응답 속도가 아니라 **브라우저 측 렉**(렌더링·스크롤·입력)을 줄이는 데 초점을 둡니다. 숨겨진 턴은 페이지에서 분리될 뿐 언제든 다시 복원할 수 있습니다.

## 왜 만들었나요

긴 AI 작업 세션은 모델 문제가 되기 전에 브라우저 성능 문제가 되는 경우가 많습니다. 개발자, 학생, 연구자, 헤비 유저는 하나의 ChatGPT 대화를 며칠 또는 몇 주씩 이어가기도 하고, 그 과정에서 페이지 안에는 수백 개의 DOM 노드, 코드 블록, 마크다운 영역, 버튼이 계속 쌓입니다. 그러면 괜찮은 기기에서도 스크롤, 입력, 탭 전환이 둔해질 수 있습니다.

ChatGPT Long Chat Slimmer는 이 병목을 줄이기 위한 작고 로컬 우선적인 도구입니다. 대화는 브라우저 세션 안에 유지하고, 네트워크 응답을 가로채지 않으며, 오래된 턴이 필요할 때는 사용자가 직접 다시 펼칠 수 있게 합니다.

## 이런 사용자에게 유용합니다

- 긴 디버깅, 설계, 코드 리뷰 세션에 ChatGPT를 쓰는 개발자.
- 하나의 대화 안에 긴 노트와 자료를 쌓는 연구자와 학생.
- Chromium 브라우저에서 입력 지연이나 스크롤 버벅임을 느끼는 ChatGPT 헤비 유저.
- 외부 전송 없이 브라우저 안에서만 동작하는 성능 도구를 원하는 사용자.

## ✨ 주요 기능

| | 기능 |
|---|---|
| ⚡ | **긴 대화 가속** — 최신 *N*개 턴만 DOM에 남기고 오래된 턴은 분리 |
| 🎚️ | **실시간 조절** — 유지 턴 수(2~200)를 바꾸면 열려 있는 탭에 즉시 반영 |
| 👀 | **언제든 복원** — 팝업이나 페이지 도크에서 `이전 더 보기`·`최신만`·`전체 보기` |
| 📊 | **상태 패널** — 전체 / 표시 / 숨김 턴 수와 DOM 경량화 비율 표시 |
| 🌐 | **이중 언어 UI** — 한국어 / English, `자동(브라우저)` 옵션 지원 |
| 🔒 | **로컬 & 비공개** — 설정은 브라우저별 로컬 저장, 외부 전송 전혀 없음 |

## ⚙️ 동작 방식

1. ChatGPT 웹 페이지에서 대화 턴 컨테이너를 찾습니다.
2. 최신 **N개** 턴만 DOM에 남기고, 더 오래된 턴은 페이지에서 분리합니다.
3. 상단 자리표시자와 접히는 우하단 도크에서 `이전 더 보기`·`최신만`·`전체 보기`를 제공합니다.
4. 유지 턴 수는 팝업과 페이지 도크에서 바로 조절되며, 설정은 브라우저별로 로컬 저장됩니다.
5. 모든 동작은 브라우저 안에서만 수행되며 외부 전송이 없습니다.

## 📦 설치 방법

1. 이 저장소를 내려받거나 클론합니다.
2. Chrome(또는 Chromium 계열 브라우저)에서 `chrome://extensions` 를 엽니다.
3. 우측 상단의 **개발자 모드**를 켭니다.
4. **압축해제된 확장 프로그램을 로드합니다**를 누릅니다.
5. 이 폴더를 선택합니다.

## 🚀 사용법

- `chatgpt.com` 또는 `chat.openai.com` 의 대화 페이지를 엽니다.
- 확장 아이콘을 눌러 유지 턴 수와 `이전 더 보기` 배치 크기를 실시간으로 조절합니다.
- 기본값은 **최근 40개 턴 유지**입니다.
- 오래된 턴을 다시 보려면 페이지 상단 자리표시자, 펼친 도크 패널, 또는 팝업에서 `이전 더 보기`를 누릅니다.
- 설정이 브라우저별로 저장되므로 Chrome, Edge, Arc 등에서 각각 다르게 맞출 수 있습니다.

## 🔧 설정 항목

| 설정 | 기본값 | 범위 | 설명 |
|---|---|---|---|
| **활성화** | `켜짐` | — | 경량화 기능 전체 토글 |

| **이전 더 보기 배치** | `20` | 2~200 | `이전 더 보기` 한 번에 펼칠 턴 개수 |
| **언어** | `자동` | 자동 / 한국어 / English | 팝업과 페이지 내 조작 UI 언어 |
| **상태 패널** | `켜짐` | — | 실시간 통계를 보여 주는 우하단 도크 표시 |

### 상태 패널 지표

`전체 턴` · `표시 중` · `숨김` · `DOM 경량화 %` — 활성 탭 기준으로 실시간 갱신됩니다.

## 🔒 개인정보 & 권한

- 모든 동작은 브라우저 안에서 실행되며 **어떤 데이터도 수집·전송하지 않습니다**.
- 네트워크 응답을 가로채지 않고, DOM 노드를 분리/재부착만 합니다.
- 요청하는 권한:
  - `storage` — 설정을 브라우저별로 로컬 저장.
  - `tabs` — 활성 ChatGPT 탭에 변경을 적용하고 상태를 읽기.

## 📝 주의점 & 한계

- ChatGPT 웹 DOM 구조는 비공개 구현이라 바뀔 수 있습니다. 구조가 크게 바뀌면 선택자 업데이트가 필요할 수 있습니다.
- 이 버전은 서버 응답 속도가 아니라 **브라우저 렌더링/스크롤/입력 렉** 완화를 목표로 합니다.
- 현재 구현은 더 안전한 DOM 기반 경량화 방식이며, 네트워크 응답을 가로채는 방식은 사용하지 **않습니다**.

## 🛠️ 유지보수 방향

이 프로젝트는 가벼운 오픈소스 유틸리티로 유지합니다. 가장 중요한 유지보수는 ChatGPT DOM 변화 추적, 개인정보 보호 원칙 유지, 백그라운드 서비스 없이 UI를 개선하는 것입니다.

가까운 우선순위:

- ChatGPT 마크업이 바뀌어도 턴 감지가 잘 버티도록 개선.
- DOM 구조 회귀를 재현할 수 있는 작은 fixture 추가.
- 팝업과 도크의 접근성, 키보드 조작 개선.
- 긴 대화에서의 실제 성능 리포트 수집.
